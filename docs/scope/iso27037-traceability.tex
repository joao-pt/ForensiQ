\section{Introdução}
% =============================================================================

Este documento apresenta a matriz de rastreabilidade entre os requisitos da norma
ISO/IEC 27037:2012 (\emph{Guidelines for identification, collection, acquisition
and preservation of digital evidence}) e as funcionalidades implementadas na
plataforma ForensiQ.

ISO/IEC 27037 é a norma internacional que estabelece práticas comuns para o
manuseamento de prova digital, desde a identificação até à preservação, com ênfase
em garantir integridade, cadeia de custódia e rastreabilidade. ForensiQ foi
concebida com base nestes princípios, incorporando controlos técnicos e
organizacionais em todas as camadas da aplicação.

% =============================================================================
\section{Matriz de Rastreabilidade}
% =============================================================================

\subsection*{Secção 5 --- Princípios gerais de manuseamento de prova digital}

\subsubsection*{§5.1 — Princípios gerais}

Implementação: Controlo de acesso baseado em papéis (RBAC) com dois perfis principais:
AGENT (recolha de prova no terreno) e EXPERT (perito forense). Tokens JWT com
rotação automática e blacklist. Endpoints protegidos por autorização granular.
Todas as operações auditadas com correlation IDs.

\subsubsection*{§5.2 — Identificação de prova digital}

Implementação: Modelo Evidence com campos obrigatórios: tipo (STORAGE\_MEDIA, USB\_DRIVE,
HARD\_DRIVE, SD\_CARD, etc.), descrição textual, número de série único,
georreferenciação GPS, e timestamp UTC registados no servidor. Cada evidência tem
ID único e código de barras gerado automaticamente.

\subsubsection*{§5.3 — Recolha e aquisição de prova}

Implementação: Registro fotográfico obrigatório com upload de imagens. Metadados GPS
capturados automaticamente via API de localização do navegador. Timestamp UTC do
servidor garante origem confiável. Exportação de PDF com cadeia de custódia completa
e evidências referenciadas.

\subsubsection*{§5.4 — Preservação de integridade}

Implementação: Hash SHA-256 calculado automaticamente no momento de criação de cada
evidência. Imutabilidade implementada em 3 camadas:
\begin{enumerate}
    \item Read-only no modelo Django após criação
    \item Validação no serializador DRF
    \item Triggers PostgreSQL que rejeitam atualizações de campos críticos
\end{enumerate}
Qualquer tentativa de modificação é registada no AuditLog.

\subsubsection*{§5.5 — Cadeia de custódia}

Implementação: Modelo ChainOfCustody com registo append-only imutável. Máquina de
estados com 7 estados: RECEIVED, IN\_EXAMINATION, EXAMINED, STORED, ARCHIVED,
DESTROYED, RELEASED. Hash encadeado tipo blockchain: cada transição referencia o
hash da anterior, impedindo inserção retroativa de registos.

\subsection*{Secção 6 --- Competência de pessoal e processo}

\subsubsection*{§6.1 — Competência do pessoal}

Implementação: Sistema de papéis diferenciados. AGENT pode criar ocorrências e
evidências, mas sem acesso a dados de outros agentes. EXPERT tem acesso completo
para análise e relatórios. Dashboard mostra apenas dados relevantes ao perfil.
Permissões aplicadas ao nível de queryset (filtragem automática por agent-id).

\subsubsection*{§6.2 — Documentação do processo}

Implementação: Exportação de PDF em conformidade com ISO, incluindo: dados da
ocorrência, lista de evidências com metadados, cadeia de custódia com transições
datadas, assinatura digital do perito e carimbo temporal. Registo de todas as
transições de estado com observações opcionais do utilizador. Auditoria completa
descarregável em formato estruturado.

\subsection*{Secção 7 --- Tipos de prova digital}

\subsubsection*{§7.1 — Dispositivos digitais}

Implementação: Modelo DigitalDevice com ficha completa: tipo (smartphone, computador,
tablet, servidor, etc.), marca, modelo, IMEI (se aplicável), estado de origem,
número de série, data de aquisição, notas técnicas. Possibilita rastreamento
completo e identificação única de cada dispositivo.

\subsubsection*{§7.2 — Suportes de armazenamento}

Implementação: Modelo Evidence com taxonomia de tipos: STORAGE\_MEDIA, USB\_DRIVE,
HARD\_DRIVE, SD\_CARD, MEMORY\_CARD, SIM\_CARD, MICRO\_SD, CLOUD\_STORAGE.
Cada tipo tem campos específicos e validações apropriadas.

\subsection*{Segurança e auditabilidade do sistema}

\subsubsection*{Auditabilidade}

Implementação: Módulo AuditLog estruturado registando: utilizador, ação, recurso,
parâmetros, timestamp, IP, correlation ID, resultado. Logs consultáveis via Django
Admin protegido e via endpoint GET com permissões apropriadas. Impossível modificar
histórico após registo.

\subsubsection*{Segurança da infraestrutura}

Implementação: CSP (Content Security Policy) restritiva. HSTS para forçar HTTPS.
Rate limiting (10 requisições/minuto para utilizadores anónimos). CSRF via token
csrftoken. select-for-update() para evitar race conditions na cadeia de custódia.
Django Admin em URL aleatória, não em /admin. Passwords com hash PBKDF2.

\subsubsection*{Proteção contra acesso não autorizado (IDOR)}

Implementação: Filtro automático de querysets por agent-id do utilizador
autenticado. Cada agente vê apenas suas ocorrências e provas. Testes de IDOR
executados para endpoints críticos. Validação ao nível do serializador confirma
que o agente tem permissão de escrita/leitura.

\section{Notas de Implementação}
% =============================================================================

\begin{itemize}
    \item \textbf{Imutabilidade multicamada:} A implementação em 3 camadas
          (aplicação + API + base de dados) garante que mesmo um atacante com
          acesso à base de dados não consegue modificar retroativamente uma evidência
          sem deixar vestígios.

    \item \textbf{Rastreabilidade granular:} Cada operação é registada com ID de
          correlação, permitindo auditar toda a cadeia de acontecimentos num processo.

    \item \textbf{Conformidade com ISO:} Embora ForensiQ não seja certificada ISO/IEC
          27037, segue os princípios e controlos definidos pela norma, em particular
          no que respeita a integridade, cadeia de custódia e competência de pessoal.

    \item \textbf{Extensibilidade:} A arquitetura permite adicionar novas camadas de
          forense digital (Química, Biologia, Física) sem comprometer a estrutura base
          de prova digital.
\end{itemize}
